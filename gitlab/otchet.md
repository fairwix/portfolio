# Отчёт по лабораторной работе "Работа с Git"

## Часть 1. Выполнение команд в терминале (по скринкасту)

### 1.1 Инициализация репозитория
Начали работу с инициализации пустого репозитория командой `git init`. Просмотрели содержимое папки с помощью `ls -la` и проверили статус через `git status` — репозиторий пуст, коммитить нечего.
![photo](photo/photo_2026-02-20%202.05.47%20PM.jpeg)

Создали файл `readme.txt` командой `touch readme.txt`:
![photo](photo/photo_2026-02-20%202.05.49%20PM.jpeg)

Отредактировали файл через `nano readme.txt` для того чтобы зафиксировать. Затем выполнили git status и git add чтобы статус файла стал отслеживаемым. Для того чтобы зафиксировать вводим команду `git commit -m "First file"` после этого он теперь хранится в репозитории. Вводим еще раз `git status` и видим, что нам пока больше нечего фиксировать, так как не было добавлено новых файлов.
![photo](photo/photo_2026-02-20%202.05.50%20PM.jpeg)

### 1.2 Работа с изменениями
После изменения файла снова проверили статус — файл отслеживается, но изменения не добавлены для коммита.
![photo](photo/photo_2026-02-20%202.05.52%20PM.jpeg)

Создали копии файла и добавили все `.txt` файлы одной командой `git add '*.txt'`.
![photo](photo/photo_2026-02-20%202.05.53%20PM.jpeg)

Зафиксировали изменения и просмотрели историю.
![photo](photo/photo_2026-02-20%202.06.04%20PM.jpeg)

### 1.3 Работа с удалённым репозиторием
Далее я создала удаленный репозиторий, выполнила `git remote add origin` и ввела свой адрес. Далее `git push -u origin master` - выгрузила все изменения.
![photo](photo/photo_2026-02-20%202.06.16%20PM.jpeg)

В браузере изменила файл `readme.txt`.
![photo](photo/photo_2026-02-20%202.06.23%20PM.jpeg)

И в консоли выполнила операцию `git pull origin master`.
![photo](photo/photo_2026-02-20%202.06.25%20PM.jpeg)

### 1.4 Работа с изменениями и отменами
Изменила файл и посмотрела различия (_Просмотр изменений через git diff_).
![photo](photo/photo_2026-02-20%202.06.26%20PM%201.jpeg)

Создала каталог и файл, проверила статус.
![photo](photo/photo_2026-02-20%202.06.28%20PM%201.jpeg)

Выполнила `git diff --staged` и `git reset`.
![photo](photo/photo_2026-02-20%202.06.31%20PM%201.jpeg)

Вернула изменения файла командой `git checkout -- readme.txt`.
![photo](photo/photo_2026-02-20%202.06.32%20PM%201.jpeg)

### 1.5 Работа с ветками
Создала ветку с помощью `git branch` и переключилась на нее. Удалила из нее все ненужные файлы и каталог, как показано в скринкасте.
![photo](photo/photo_2026-02-20%202.06.36%20PM%202.jpeg)
![photo](photo/photo_2026-02-20%202.06.37%20PM%201.jpeg)

Вернулась в ветку `master` и осуществила слияние этих веток с помощью `git merge`.
![photo](photo/photo_2026-02-20%202.06.39%20PM.jpeg)

Удалила ветку и выгрузила последние изменения.
![photo](photo/photo_2026-02-20%202.06.40%20PM.jpeg)
![photo](photo/photo_2026-02-20%202.06.42%20PM.jpeg)

## Часть 2. Выполнение команд в VS Code

### 2.1 Подготовка
Открыла VS Code, установила расширение **Git History** для удобного просмотра истории коммитов. Затем открыла папку проекта: `File → Open Folder` → выбрала `my_project`.
![photo](photo/IMAGE%202026-02-20%2011%2017%2045%20PM.jpg)

Репозиторий уже был инициализирован. При необходимости инициализации используется палитра команд `Cmd+Shift+P` → `Git: Initialize Repository`.
![photo](photo/IMAGE%202026-02-20%2011%2018%2043%20PM.jpg)
![photo](photo/IMAGE%202026-02-20%2011%2018%2054%20PM.jpg)

Проверила наличие скрытой папки `.git` в Finder.
![photo](photo/IMAGE%202026-02-20%2011%2019%2041%20PM.jpg)

### 2.2 Создание и коммит файла
В explorer создала файл `readme2.txt`, написала текст и сохранила (`Cmd+S`).
![photo](photo/IMAGE%202026-02-20%2011%2020%2040%20PM.jpg)

В панели Source Control файл появился с пометкой **U** (untracked). Для добавления в staging area нажала "+" (Stage Changes).
![photo](photo/IMAGE%202026-02-20%2011%2021%2017%20PM.jpg)

Теперь файл переместился в `staged changes`.
![photo](photo/IMAGE%202026-02-20%2011%2021%2032%20PM.jpg)

Ввела сообщение и нажала Commit (`Cmd+Enter`). После коммита раздел `staged changes` исчезает.
![photo](photo/IMAGE%202026-02-20%2011%2021%2045%20PM.jpg)
![photo](photo/IMAGE%202026-02-20%2011%2021%2058%20PM.jpg)

### 2.3 Изменение файла
Если мы хотим изменить файл, то просто изменяем его и сохраняем. Изменила файл — появилась пометка **M** (modified). Нажимаю + рядом с файлом, чтобы застейджить изменение.
![photo](photo/IMAGE%202026-02-20%2011%2022%2039%20PM.jpg)

Ввожу сообщение и нажимаю `commit`.
![photo](photo/IMAGE%202026-02-20%2011%2022%2052%20PM.jpg)

### 2.4 Просмотр истории
Благодаря расширению Git History в панели Source Control отображается граф коммитов.
